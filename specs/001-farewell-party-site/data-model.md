# Data Model: Farewell Party Website

**Date**: 2026-04-09  
**Scope**: Entity definitions, relationships, validation rules, and state transitions for all data structures in the farewell party website.

---

## 1. Navigation Destination

**Definition**: A labeled destination available from the full-screen navigation overlay, with URL route and display order.

**Entity Name**: `NavigationDestination`

**Fields**:
- `id` (string): Unique identifier (e.g., "home", "rsvp", "messages")
- `label` (string): Display text for menu (e.g., "Home", "Are You Coming?", "Drop a Message")
- `route` (string): URL path (e.g., "/", "/are-you-coming", "/drop-a-message")
- `description` (string, optional): Extended copy for menu item
- `order` (number): Display order in menu (lower first)
- `icon` (string, optional): Icon identifier for future icon rendering

**Validation Rules**:
- `id` is required, lowercase alphanumeric
- `label` is required, 1–50 characters
- `route` is required and must start with `/`
- `order` is required, integer ≥ 0

**Relationships**:
- Referenced by Navigation Overlay component
- Stateless; no temporal changes

**Example**:
```typescript
{
  id: "rsvp",
  label: "Are You Coming?",
  route: "/are-you-coming",
  description: "Confirm your attendance",
  order: 1,
  icon: "check-circle"
}
```

## 2. Home Scene Section

**Definition**: A defined stage of the scrolling Home page experience with narrative purpose, visual content trigger, and relative position in the sequence.

**Entity Name**: `HomeSceneSection`

**Fields**:
- `id` (string): Unique identifier (e.g., "opening", "characters", "boxes", "van", "title", "closing")
- `title` (string): Internal name for reference
- `navLabel` (string, optional): Display label if this section has UI control
- `triggerOffset` (number): Scroll position (px from top) where animation begins
- `duration` (number): Animation duration (ms)
- `content` (string, optional): Narrative or descriptive text for the section
- `isTitleTrigger` (boolean): If true, this is the title introduction moment
- `isFinalComposition` (boolean): If true, this is the closing pixel-art area
- `assets` (string[]): List of image asset filenames to load for this section
- `order` (number): Sequence order (lower first)

**Validation Rules**:
- `id` is required, lowercase alphanumeric
- `title` is required, 1–100 characters
- `triggerOffset` is required, integer ≥ 0
- `duration` is required, integer > 0
- `order` is required, integer ≥ 0
- Only one section can have `isTitleTrigger: true`
- Only one section can have `isFinalComposition: true`

**Relationships**:
- HomePage component iterates over ordered list
- Each section is rendered by HomeScene component with animation trigger

**State Transitions**:
- Inactive → Active (when scroll position crosses `triggerOffset`)
- Active → Completed (when animation duration elapses)
- Completed → Inactive (if scroll returns above `triggerOffset`)

**Example**:
```typescript
{
  id: "title",
  title: "Title Introduction",
  triggerOffset: 1200,
  duration: 800,
  content: "The title appears here as the farewell moment",
  isTitleTrigger: true,
  isFinalComposition: false,
  assets: ["title-text.png"],
  order: 4
}
```

## 3. RSVP Submission

**Definition**: A guest attendance confirmation record containing the submitted name, submission metadata, and confirmation status.

**Entity Name**: `RSVPSubmission`

**Fields**:
- `id` (string, UUID): Unique identifier (generated by Supabase)
- `name` (string): Guest's submitted name
- `confirmedAt` (timestamp): Submission timestamp (should be now())
- `createdAt` (timestamp): Record creation timestamp
- `status` (enum): "pending" | "confirmed" (always "confirmed" for initial implementation)

**Validation Rules**:
- `name` is required, trimmed, 1–100 characters, non-empty
- `confirmedAt` is set to current server time on submission
- Status defaults to "confirmed" (no moderation queue initially)

**Relationships**:
- Stored in Supabase `guest_rsvps` table
- No foreign key relationships (independent of messages)
- Can have duplicate names (per spec edge case)

**State Transitions**:
- New → Submitted (POST /api/guest-rsvp)
- Submitted → Confirmed (immediately, no approval workflow)
- Persisted in database; read-only after creation

**Example**:
```typescript
{
  id: "e3a8b2c1-4d5e-11eb-ae93-0242ac120002",
  name: "Alice Johnson",
  confirmedAt: "2026-04-09T15:30:45Z",
  createdAt: "2026-04-09T15:30:45Z",
  status: "confirmed"
}
```

## 4. Guest Message Submission

**Definition**: A public guest message record containing the guest name, message text, and persistence metadata for rendering on the message wall.

**Entity Name**: `GuestMessageSubmission`

**Fields**:
- `id` (string, UUID): Unique identifier (generated by Supabase)
- `guestName` (string): Guest's name (author of message)
- `message` (string): Short farewell message text
- `approved` (boolean): Moderation flag (always true initially, extensible for future approval workflow)
- `createdAt` (timestamp): Submission timestamp

**Validation Rules**:
- `guestName` is required, trimmed, 1–100 characters, non-empty
- `message` is required, trimmed, 1–500 characters, non-empty
- `approved` defaults to true
- Message must not contain only whitespace

**Relationships**:
- Stored in Supabase `guest_messages` table
- No foreign key to RSVP (messages are independent of attendance confirmation)
- Multiple messages allowed per guest

**State Transitions**:
- New → Submitted (POST /api/guest-message)
- Submitted → Approved (immediately; defaults to true)
- Persisted in database; read-only after creation
- Visible on message wall only if `approved: true`

**Example**:
```typescript
{
  id: "a1b2c3d4-5e6f-47g8-9h8i-0j7k6l5m4n3o",
  guestName: "Bob Chen",
  message: "What an adventure! We'll miss you all. 🚐",
  approved: true,
  createdAt: "2026-04-09T16:45:20Z"
}
```

## 5. Message Card (Rendered)

**Definition**: The visual representation of a guest message in the public message wall, including display properties and layout behavior.

**Entity Name**: `MessageCard`

**Fields**:
- `id` (string, UUID): Reference to GuestMessageSubmission.id
- `guestName` (string): Guest name from submission
- `message` (string): Message text from submission
- `displayRotation` (number): CSS rotation in degrees (±1–3 for playful effect)
- `displayColor` (string, optional): Tailwind color class for card background
- `createdAt` (timestamp): From submission
- `displayOrder` (number, optional): Grid placement order

**Validation Rules**:
- All fields derived from GuestMessageSubmission; no validation needed at card level
- `displayRotation` is pseudo-random per card instance (not persisted)
- `displayColor` is optional; defaults to neutral tone

**Relationships**:
- 1:1 with GuestMessageSubmission
- Rendered by GuestMessageCard component
- Collected in GuestMessagesList (Grid layout)

**Example**:
```typescript
{
  id: "a1b2c3d4-5e6f-47g8-9h8i-0j7k6l5m4n3o",
  guestName: "Bob Chen",
  message: "What an adventure! We'll miss you all. 🚐",
  displayRotation: 2.5,
  displayColor: "bg-yellow-100",
  createdAt: "2026-04-09T16:45:20Z",
  displayOrder: 3
}
```

## 6. Form Input State (Ephemeral)

**Definition**: Temporary client-side state for form inputs, validation feedback, and submission progress.

**Entity Name**: `FormInputState`

**Variants**:
- `RSVPFormState`: { name: string; isValidating: boolean; error: string | null; isSubmitting: boolean; isSuccess: boolean; successMessage: string }
- `MessageFormState`: { guestName: string; message: string; isValidating: boolean; errors: { guestName?: string; message?: string }; isSubmitting: boolean; isSuccess: boolean; successMessage: string }

**Validation Rules**:
- `isValidating` is true during client-side validation check
- `isSubmitting` is true during API request
- `error` or `errors` populated only if validation fails
- `isSuccess` is true post-submission, auto-clears after 3 seconds
- Form values are persisted to localStorage if submission fails (for recovery)

**Relationships**:
- Local to component (GuestListForm, GuestMessageForm)
- Stored in component state or custom hook
- Persisted to localStorage by siteStorage service

**Example**:
```typescript
// RSVPFormState
{
  name: "Alice Johnson",
  isValidating: false,
  error: null,
  isSubmitting: false,
  isSuccess: true,
  successMessage: "Your RSVP is confirmed! See you soon."
}

// MessageFormState
{
  guestName: "Bob Chen",
  message: "What an adventure!",
  isValidating: false,
  errors: {},
  isSubmitting: false,
  isSuccess: true,
  successMessage: "Your message has been posted!"
}
```

## 7. Navigation Overlay State (Ephemeral)

**Definition**: Temporary overlay visibility and focus state, managed at app level.

**Entity Name**: `NavigationOverlayState`

**Fields**:
- `isOpen` (boolean): Overlay visibility toggle
- `scrollPositionBeforeOpen` (number): Saved scroll Y position for recovery after close
- `focusedDestinationIndex` (number, optional): Current keyboard navigation index

**Validation Rules**:
- `isOpen` defaults to false
- `scrollPositionBeforeOpen` is captured when overlay opens
- When overlay closes, page scrolls back to saved position (unless user navigated to new route)

**Relationships**:
- Stored in custom hook (useNavigationOverlay)
- Persisted to localStorage under key `farewell-party__nav-overlay-*`
- Managed by PersistentNavigation component

**Example**:
```typescript
{
  isOpen: true,
  scrollPositionBeforeOpen: 1250,
  focusedDestinationIndex: 0
}
```

## 8. Empty States & Loading States

**Definition**: Placeholder content when no data is available or data is loading.

**Entity Name**: `UIState`

**Variants**:
- `EmptyState`: { type: "no-messages" | "no-rsvps"; message: string; subtext: string }
- `LoadingState`: { isLoading: boolean; message: string }
- `ErrorState`: { error: string; retryAction: () => void }

**Examples**:

**No Messages Empty State**:
```typescript
{
  type: "no-messages",
  message: "No messages yet",
  subtext: "Be the first to leave a farewell message!"
}
```

**Message Loading State**:
```typescript
{
  isLoading: true,
  message: "Loading messages..."
}
```

**API Error State**:
```typescript
{
  error: "Failed to submit message. Please check your connection and try again.",
  retryAction: () => submitMessage()
}
```

## Summary: Entity Relationships

```
NavigationDestination (config) → referenced by Navigation Overlay
HomeSceneSection (config) → rendered by HomePage, HomeScene
RSVPSubmission (persisted) → stored in Supabase guest_rsvps table
GuestMessageSubmission (persisted) → stored in Supabase guest_messages table
MessageCard (derived) → 1:1 from GuestMessageSubmission, rendered by GuestMessageCard
FormInputState (ephemeral) → local to form components
NavigationOverlayState (ephemeral) → managed at app level, persisted to localStorage
UIState (derived) → rendered conditionally by pages/messages
```

All entities are typed in TypeScript and contracts are defined in `/specs/001-farewell-party-site/contracts/` for reference during implementation.
